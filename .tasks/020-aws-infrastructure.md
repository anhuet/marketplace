## Description

Provision the AWS infrastructure required to run the Marketplace backend in a staging/production environment. This includes: an AWS RDS PostgreSQL instance (with the correct parameter group for potential PostGIS extension), an S3 bucket for listing image storage, and a deployment target for the Node.js backend (EC2 or ECS Fargate). All infrastructure should be reproducible via IaC (CloudFormation, CDK, or Terraform) and all secrets stored in AWS Secrets Manager or Parameter Store.

## Acceptance Criteria

- [ ] AWS RDS PostgreSQL instance provisioned (db.t3.micro for staging) in a private VPC subnet with security group restricting access to the backend compute layer only
- [ ] Prisma migrations can be applied to the RDS instance from the CI/CD pipeline or a migration runner
- [ ] S3 bucket created with appropriate CORS policy for image uploads from the mobile app and public-read ACL (or presigned URL strategy) for image serving
- [ ] Backend application deployed and reachable at a stable HTTPS endpoint (EC2 with Nginx reverse proxy, or ECS Fargate behind an ALB)
- [ ] All environment variables (DB connection string, JWT secret, S3 bucket name) are stored in AWS Secrets Manager and injected into the runtime environment — not committed to source control
- [ ] Infrastructure configuration is documented in `docs/technical/ARCHITECTURE.md` with a network diagram or textual description of the VPC, subnets, and security groups

## Technical Notes

- RDS instance class: start with `db.t3.micro` for staging; plan for `db.t3.small` or larger in production. Enable automated backups (7-day retention minimum).
- If PostGIS extension is needed (task #007 upgrade path), verify the RDS parameter group allows `shared_preload_libraries = 'postgis'` — this requires a db instance restart.
- ECS Fargate vs EC2: Fargate is preferred for lower operational overhead; EC2 is acceptable if cost is a constraint. Document the decision in `docs/technical/DECISIONS.md`.
- S3 image serving: presigned URLs (short TTL, generated by the backend) are more secure than public-read ACL — recommend this approach but flag the added backend complexity.
- CI/CD pipeline for backend deployments is a separate task that can be raised as a follow-on backlog item once this infrastructure is stable.
